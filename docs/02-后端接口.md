# 后端接口说明

本文记录 FreshFoodSystem-v2 当前已实现和计划实现的后端接口。

统一返回格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

## 1. 基础接口

### 健康检查

```http
GET /api/health
```

用于确认后端服务是否正常运行。

## 2. 分类接口

### 查询启用分类

```http
GET /api/categories
```

返回 `status = 1` 的分类，并按 `sort_order` 和 `id` 升序排列。

## 3. 商品接口

### 查询上架商品

```http
GET /api/products/onsale
```

返回 `status = 1` 的商品。

### 按分类查询商品

```http
GET /api/products/category/{categoryId}
```

示例：

```http
GET /api/products/category/1
```

### 查询热销商品

```http
GET /api/products/hot
```

返回 `is_hot = 1` 且 `status = 1` 的商品。

### 查询新品商品

```http
GET /api/products/new
```

返回 `is_new = 1` 且 `status = 1` 的商品。

### 查询商品详情

```http
GET /api/products/{id}
```

示例：

```http
GET /api/products/1
```

## 4. 用户接口

### 用户登录或自动注册

```http
POST /api/users/login?openid={openid}&nickName={nickName}&avatarUrl={avatarUrl}
```

说明：

- 如果 `openid` 不存在，则创建用户。
- 如果 `openid` 已存在，则更新昵称、头像和最后登录时间。
- `nickName` 和 `avatarUrl` 可以为空。
- 登录成功后返回 `token`、`tokenType` 和 `expiresIn`。

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "tokenType": "Bearer",
    "token": "xxxxx",
    "expiresIn": 7200,
    "user": {
      "id": 1,
      "openid": "test_openid_001"
    }
  }
}
```

示例：

```http
POST /api/users/login?openid=test_openid_001&nickName=测试用户&avatarUrl=/images/user.png
```

### 查询当前登录用户

```http
GET /api/users/current
Authorization: Bearer {token}
```

### 查询全部用户

```http
GET /api/users
```

当前主要用于开发测试。

### 根据 ID 查询用户

```http
GET /api/users/{id}
```

示例：

```http
GET /api/users/1
```

### 根据 openid 查询用户

```http
GET /api/users/openid/{openid}
```

示例：

```http
GET /api/users/openid/demo_openid_user_001
```

## 5. 地址接口

### 查询用户地址列表

```http
GET /api/addresses/user/{userId}
```

默认地址会排在前面。

### 查询当前用户地址列表

```http
GET /api/addresses/current
Authorization: Bearer {token}
```

### 新增地址

```http
POST /api/addresses
Content-Type: application/json
```

请求体示例：

```json
{
  "userId": 1,
  "receiverName": "王同学",
  "receiverPhone": "13800000003",
  "province": "广东省",
  "city": "广州市",
  "district": "天河区",
  "detailAddress": "测试小区8栋808",
  "label": "家",
  "isDefault": 1
}
```

说明：

- `isDefault = 1` 表示设为默认地址。
- 如果新增默认地址，后端会自动把同一用户的旧默认地址改为普通地址。

### 新增当前用户地址

```http
POST /api/addresses/current
Authorization: Bearer {token}
Content-Type: application/json
```

### 修改地址

```http
PUT /api/addresses/{id}
Content-Type: application/json
```

请求体示例：

```json
{
  "receiverName": "王同学",
  "label": "公司",
  "detailAddress": "测试小区10栋1001"
}
```

### 设置默认地址

```http
PUT /api/addresses/{id}/default
```

说明：

- 设置当前地址为默认地址。
- 同一用户原来的默认地址会被自动改为非默认地址。

### 删除地址

```http
DELETE /api/addresses/{id}
```

## 6. 购物车接口

```http
GET    /api/carts/user/{userId}
GET    /api/carts/user/{userId}/selected
POST   /api/carts
PUT    /api/carts/{id}/quantity
PUT    /api/carts/{id}/selected
DELETE /api/carts/{id}
DELETE /api/carts/user/{userId}
```

### 查询用户购物车

```http
GET /api/carts/user/{userId}
```

### 查询用户已选中购物车

```http
GET /api/carts/user/{userId}/selected
```

### 加入购物车

```http
POST /api/carts?userId={userId}&productId={productId}&quantity={quantity}
```

如果同一用户已加入同一商品，后端会累加数量。

### 修改购物车数量

```http
PUT /api/carts/{id}/quantity?quantity={quantity}
```

如果 `quantity <= 0`，后端会删除该购物车项。

### 修改购物车选中状态

```http
PUT /api/carts/{id}/selected?selected={selected}
```

`selected = 1` 表示选中，`selected = 0` 表示未选中。

### 删除购物车项

```http
DELETE /api/carts/{id}
```

### 清空用户购物车

```http
DELETE /api/carts/user/{userId}
```

### 当前用户购物车

```http
GET /api/carts/current
GET /api/carts/current/selected
POST /api/carts/current?productId={productId}&quantity={quantity}
DELETE /api/carts/current
Authorization: Bearer {token}
```

## 7. 订单接口

### 查询用户订单

```http
GET /api/orders/user/{userId}
```

### 按状态查询用户订单

```http
GET /api/orders/user/{userId}/status/{orderStatus}
```

状态说明：

| 值 | 含义 |
| --- | --- |
| `0` | 待付款 |
| `1` | 待发货 |
| `2` | 待收货 |
| `3` | 已完成 |
| `4` | 已取消 |

### 查询订单详情

```http
GET /api/orders/{id}
```

订单详情会返回订单主表信息和 `orderItems` 订单明细。

### 从购物车创建订单

```http
POST /api/orders?userId={userId}&addressId={addressId}&userRemark={userRemark}
```

创建订单时后端会执行：

- 读取用户购物车中 `selected = 1` 的商品。
- 校验商品是否上架、库存是否足够。
- 计算订单金额。
- 写入 `orders` 订单主表。
- 写入 `order_item` 订单明细。
- 扣减商品库存。
- 删除已结算的购物车项。

### 当前用户订单

```http
GET /api/orders/current
GET /api/orders/current/status/{orderStatus}
POST /api/orders/current?addressId={addressId}&userRemark={userRemark}
Authorization: Bearer {token}
```

### 模拟支付

```http
PUT /api/orders/{id}/mock-pay
```

把订单从 `待付款` 改为 `待发货`，并记录支付方式为模拟支付。

### 发货

```http
PUT /api/orders/{id}/ship?trackingNo={trackingNo}
```

把订单从 `待发货` 改为 `待收货`，并记录发货时间和运单号。

### 确认收货

```http
PUT /api/orders/{id}/confirm-receive
```

把订单从 `待收货` 改为 `已完成`，并记录确认收货时间。

### 取消订单

```http
PUT /api/orders/{id}/cancel
```

当前只允许取消 `待付款` 订单。取消后会恢复商品库存。

## 8. 类型映射约定

后端实体类型必须和数据库字段类型保持一致。

| MySQL 类型 | Java 类型 |
| --- | --- |
| `bigint` | `Long` |
| `int` | `Integer` |
| `tinyint` | `Byte` |
| `varchar` | `String` |
| `text` | `String` |
| `decimal(10,2)` | `BigDecimal` |
| `datetime` | `LocalDateTime` |

## 9. 常见注解说明

| 注解 | 含义 |
| --- | --- |
| `@RestController` | 声明控制器，返回值自动转成 JSON |
| `@RequestMapping` | 给一组接口设置公共路径 |
| `@GetMapping` | 处理 GET 请求，通常用于查询 |
| `@PostMapping` | 处理 POST 请求，通常用于新增或提交 |
| `@PutMapping` | 处理 PUT 请求，通常用于修改 |
| `@DeleteMapping` | 处理 DELETE 请求，通常用于删除 |
| `@PathVariable` | 从 URL 路径中取参数 |
| `@RequestParam` | 从查询参数或表单参数中取值 |
| `@RequestBody` | 从 JSON 请求体中取值 |
| `@Validated` | 开启方法级参数校验 |
| `@Valid` | 开启对象校验 |
| `@Service` | 声明业务逻辑层 |
| `@Transactional` | 声明事务，保证多步数据库操作一起成功或失败 |
| `@Entity` | 声明 JPA 实体类 |
| `@Table` | 指定实体类对应的数据表 |
| `@Column` | 指定字段和数据库列的映射 |
