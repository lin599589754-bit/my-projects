# 后端接口说明

本文记录 FreshFoodSystem-v2 当前后端接口。接口统一返回 `ApiResponse<T>`。

## 1. 通用约定

### 1.1 基础地址

```text
http://localhost:8080
```

### 1.2 统一返回格式

成功：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

失败：

```json
{
  "code": 400,
  "message": "错误原因",
  "data": null
}
```

### 1.3 认证方式

除公开接口外，请求需要携带：

```http
Authorization: Bearer {token}
```

### 1.4 常用错误码

| HTTP 状态码 | JSON code | 含义 |
| --- | --- | --- |
| `200` | `200` | 请求成功 |
| `400` | `400` | 参数错误或业务规则不满足 |
| `401` | `401` | 未登录或 token 无效 |
| `403` | `403` | 无权限访问 |
| `404` | `404` | 数据不存在 |
| `405` | `405` | 请求方式错误 |
| `500` | `500` | 未知服务器错误 |

## 2. 公开接口

### 2.1 健康检查

```http
GET /api/health
```

用于确认后端服务是否正常运行。

### 2.2 用户登录或自动注册

```http
POST /api/users/login
Content-Type: application/json
```

请求体：

```json
{
  "openid": "test-openid",
  "nickName": "测试用户",
  "avatarUrl": "https://example.com/avatar.png"
}
```

说明：

- `openid` 必填。
- `nickName` 为空时默认使用 `微信用户`。
- `avatarUrl` 为空时默认使用空字符串。
- 如果用户不存在，则自动创建。
- 如果用户已存在，则更新昵称、头像和最后登录时间。

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "tokenType": "Bearer",
    "token": "xxxxx",
    "expiresIn": 7200,
    "user": {
      "id": 1,
      "openid": "test-openid",
      "nickName": "测试用户"
    }
  }
}
```

### 2.3 查询启用分类

```http
GET /api/categories
```

返回 `status = 1` 的分类，并按 `sort_order` 和 `id` 升序排列。

### 2.4 查询商品

```http
GET /api/products/onsale
GET /api/products/category/{categoryId}
GET /api/products/hot
GET /api/products/new
GET /api/products/{id}
```

说明：

- `onsale`：查询上架商品。
- `category/{categoryId}`：按分类查询上架商品。
- `hot`：查询热销商品。
- `new`：查询新品商品。
- `{id}`：查询商品详情。

## 3. 用户接口

### 3.1 查询当前登录用户

```http
GET /api/users/current
Authorization: Bearer {token}
```

### 3.2 根据 ID 查询用户

```http
GET /api/users/{id}
Authorization: Bearer {token}
```

说明：

- 只能查询当前 token 对应的用户。
- 查询其他用户会返回 `403`。

### 3.3 当前禁止普通用户访问的用户接口

```http
GET /api/users
GET /api/users/openid/{openid}
```

说明：

- 当前版本没有管理员角色。
- 这两个接口涉及用户数据枚举，已在安全配置中禁止访问。
- 普通用户访问会返回 `403 无权限访问`。

## 4. 地址接口

### 4.1 查询指定用户地址

```http
GET /api/addresses/user/{userId}
Authorization: Bearer {token}
```

说明：

- 只能查询当前 token 对应用户的地址。
- 默认地址会排在前面。

### 4.2 查询当前用户地址

```http
GET /api/addresses/current
Authorization: Bearer {token}
```

### 4.3 新增地址

```http
POST /api/addresses
Authorization: Bearer {token}
Content-Type: application/json
```

请求体：

```json
{
  "userId": 1,
  "receiverName": "王同学",
  "receiverPhone": "13800000003",
  "province": "广东省",
  "city": "广州市",
  "district": "天河区",
  "detailAddress": "测试小区8栋808",
  "label": "家",
  "isDefault": 1
}
```

说明：

- `isDefault = 1` 表示设为默认地址。
- 如果新增默认地址，后端会自动把同一用户旧默认地址改为普通地址。

### 4.4 新增当前用户地址

```http
POST /api/addresses/current
Authorization: Bearer {token}
Content-Type: application/json
```

请求体不需要 `userId`，用户 ID 从 token 中获取。

### 4.5 修改地址

```http
PUT /api/addresses/{id}
Authorization: Bearer {token}
Content-Type: application/json
```

请求体：

```json
{
  "receiverName": "王同学",
  "receiverPhone": "13800000003",
  "province": "广东省",
  "city": "深圳市",
  "district": "南山区",
  "detailAddress": "科技园1号",
  "label": "公司",
  "isDefault": 0
}
```

### 4.6 修改当前用户地址

```http
PUT /api/addresses/current/{id}
Authorization: Bearer {token}
Content-Type: application/json
```

### 4.7 设置默认地址

```http
PUT /api/addresses/{id}/default
Authorization: Bearer {token}
```

### 4.8 删除地址

```http
DELETE /api/addresses/{id}
Authorization: Bearer {token}
```

## 5. 购物车接口

### 5.1 查询指定用户购物车

```http
GET /api/carts/user/{userId}
GET /api/carts/user/{userId}/selected
Authorization: Bearer {token}
```

说明：

- 只能查询当前 token 对应用户的购物车。
- `selected` 表示只查询选中的购物车项。

### 5.2 查询当前用户购物车

```http
GET /api/carts/current
GET /api/carts/current/selected
Authorization: Bearer {token}
```

### 5.3 加入购物车

```http
POST /api/carts
Authorization: Bearer {token}
Content-Type: application/json
```

请求体：

```json
{
  "userId": 1,
  "productId": 1,
  "quantity": 2
}
```

说明：

- 只能给当前 token 对应用户加入购物车。
- 同一用户同一商品会累加数量。
- 商品不存在、商品下架、库存不足时返回业务错误。

### 5.4 加入当前用户购物车

```http
POST /api/carts/current?productId={productId}&quantity={quantity}
Authorization: Bearer {token}
```

说明：

- 用户 ID 从 token 中获取。
- `quantity` 可以省略，默认按 1 处理。

### 5.5 修改购物车数量

```http
PUT /api/carts/{id}/quantity?quantity={quantity}
Authorization: Bearer {token}
```

说明：

- 只能修改当前用户自己的购物车项。
- `quantity <= 0` 时删除该购物车项。

### 5.6 修改购物车选中状态

```http
PUT /api/carts/{id}/selected?selected={selected}
Authorization: Bearer {token}
```

说明：

- `selected = 1` 表示选中。
- `selected = 0` 表示未选中。

### 5.7 删除购物车项

```http
DELETE /api/carts/{id}
Authorization: Bearer {token}
```

### 5.8 清空购物车

```http
DELETE /api/carts/user/{userId}
DELETE /api/carts/current
Authorization: Bearer {token}
```

## 6. 订单接口

### 6.1 查询指定用户订单

```http
GET /api/orders/user/{userId}
GET /api/orders/user/{userId}/status/{orderStatus}
Authorization: Bearer {token}
```

说明：

- 只能查询当前 token 对应用户的订单。
- `orderStatus` 范围为 `0` 到 `4`。

### 6.2 查询当前用户订单

```http
GET /api/orders/current
GET /api/orders/current/status/{orderStatus}
Authorization: Bearer {token}
```

### 6.3 查询订单详情

```http
GET /api/orders/{id}
Authorization: Bearer {token}
```

说明：

- 只能查询当前用户自己的订单。
- 订单存在时返回订单主表和 `orderItems` 订单明细。
- 订单不存在时返回 `404 订单不存在`。

### 6.4 从购物车创建订单

```http
POST /api/orders
Authorization: Bearer {token}
Content-Type: application/json
```

请求体：

```json
{
  "userId": 1,
  "addressId": 1,
  "userRemark": "尽快送达"
}
```

创建订单时后端会：

- 读取当前用户选中的购物车项。
- 校验商品是否上架。
- 校验库存是否足够。
- 校验收货地址是否属于当前用户。
- 计算订单金额。
- 写入订单主表。
- 写入订单明细。
- 扣减商品库存。
- 删除已结算购物车项。

### 6.5 当前用户创建订单

```http
POST /api/orders/current?addressId={addressId}&userRemark={userRemark}
Authorization: Bearer {token}
```

说明：

- 用户 ID 从 token 中获取。
- `userRemark` 可以省略。

### 6.6 模拟支付

```http
PUT /api/orders/{id}/mock-pay
Authorization: Bearer {token}
```

说明：

- 只能操作当前用户自己的订单。
- 只能把 `待付款` 订单改为 `待发货`。
- 支付方式记录为模拟支付。

### 6.7 发货

```http
PUT /api/orders/{id}/ship?trackingNo={trackingNo}
Authorization: Bearer {token}
```

说明：

- 当前版本没有管理员角色，该接口已禁止普通用户访问。
- 后续做管理后台时，可以改为仅管理员可访问。

### 6.8 确认收货

```http
PUT /api/orders/{id}/confirm-receive
Authorization: Bearer {token}
```

说明：

- 只能操作当前用户自己的订单。
- 只能把 `待收货` 订单改为 `已完成`。

### 6.9 取消订单

```http
PUT /api/orders/{id}/cancel
Authorization: Bearer {token}
```

说明：

- 只能操作当前用户自己的订单。
- 只能取消 `待付款` 订单。
- 取消后会恢复商品库存。

## 7. 类型映射约定

后端实体类型必须和数据库字段类型保持一致。

| MySQL 类型 | Java 类型 |
| --- | --- |
| `bigint` | `Long` |
| `int` | `Integer` |
| `tinyint` | `Byte` |
| `varchar` | `String` |
| `text` | `String` |
| `decimal(10,2)` | `BigDecimal` |
| `datetime` | `LocalDateTime` |

## 8. 常见注解说明

| 注解 | 含义 |
| --- | --- |
| `@RestController` | 声明控制器，返回值自动转成 JSON |
| `@RequestMapping` | 给一组接口设置公共路径 |
| `@GetMapping` | 处理 GET 请求，通常用于查询 |
| `@PostMapping` | 处理 POST 请求，通常用于新增或提交 |
| `@PutMapping` | 处理 PUT 请求，通常用于修改 |
| `@DeleteMapping` | 处理 DELETE 请求，通常用于删除 |
| `@PathVariable` | 从 URL 路径中取参数 |
| `@RequestParam` | 从查询参数或表单参数中取值 |
| `@RequestBody` | 从 JSON 请求体中取值 |
| `@Validated` | 开启方法级参数校验 |
| `@Valid` | 开启对象校验 |
| `@Service` | 声明业务逻辑层 |
| `@Transactional` | 声明事务，保证多步数据库操作一起成功或失败 |
| `@Entity` | 声明 JPA 实体类 |
| `@Table` | 指定实体类对应的数据表 |
| `@Column` | 指定字段和数据库列的映射 |
